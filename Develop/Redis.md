# 레디스
## 레디스란
- Key, Value 구조의 비정형 데이터를 저장하고 관리하기 위한 오픈 소스 기반의 비관계형 데이터 베이스 관리 시스템
- 데이터 베이스, 캐시, 메세지 브로커로 사용되며 인메모리 데이터 구조를 가진 저장소이다.
- Redis는 HTTP 서버처럼 동작하는 게 아니라, TCP 기반 서버

## 왜 사용할까?
- 데이터 베이스가 있는데 굳이 인메모리 데이터 베이스를 왜 사용할까?
- 데이터 베이스는 접근하는데 사용자가 많을 수록 부하가 많아져 느리다.
- 그래서 자주 사용하는 정보는 데이터 베이스를 거치지 않고 인메모리 데이터 베이스를 사용하여 캐싱 처리를 위해 사용한다.
- 이 외에도 세션 관리, 실시간 분석, 메시지 큐로 써의 역할도 한다.

## 레디스 특징
- key-value 형식의 비관계형 데이터베이스
- In Memory 데이터 베이스
- In Memory 데이터 베이스이지만, 데이터의 영속성을 보장 가능 (RDBMS 만큼의 정합성과 영속성 보장은 ❌)
- 싱글 쓰레드 방식으로 동작

## 레디스의 영속성 
- In Memory 데이터 베이스는 Application이 종료되는 순간 데이터가 날아간다.
- 하지만 Redis는 다음의 두 가지 방식으로 영속성을 보장 즉 비휘발성을 보장한다.
    1. RDB(SnapShotting) 방식
        - 이 방식은 주기적으로 Redis가 디스크에 접근하여 Redis의 상태를 옮겨서 저장한다.
        - 이 방식은 Redis의 데이터를 옮기는 동안 Redis가 Blocking 될 수 있으며 데이터의 손실 가능성이 존재
    2. AOF 방식
        - Redis에 작용하는 모든 Write/Update 연산을 별도의 로그 파일에 기록하는 방식
        - 로그 파일을 복구하는데 시간이 걸리며 로그 파일이 계속 커지면서 많은 공간을 차지 할 수 있음
- 일반적으로 두 방식을 혼합해서 사용

## 레디스 동작 원리
Redis는 TCP로 들어온 명령을 이벤트 루프에서 감지하고,
명령어를 파싱한 뒤,
메모리에 있는 key-value 자료구조를 직접 조작하고,
그 결과를 클라이언트에게 반환한다.

명령 실행은 기본적으로 한 번에 하나씩 처리되기 때문에 단일 명령의 원자성이 보장된다.

TTL은 key별 만료 시간을 별도로 관리하고,
영속화는 RDB 스냅샷 또는 AOF 명령 로그 방식으로 처리하며,
복제는 Master의 변경 명령을 Replica가 따라 실행하는 방식으로 동작한다.

- Redis는 명령어 처리를 기본적으로 단일 스레드 이벤트 루프 방식으로 처리
- 즉, 여러 클라이언트가 동시에 요청을 보내도 Redis 내부에서는 명령을 빠르게 하나씩 처리하는 구조 -> Redis 명령 하나하나는 원자적으로 처리 -> 조회수 증가, 재고 차감, 락 획득 같은 작업에 사용

## 자료구조
- 기본적으로 Key-Value 기반의 데이터 저장소로, Value로는 단순히 String 뿐만 아니라 다양한 데이터 타입을 지원
    - String
        - 일반 문자열 (최대 512MB)
        - 단순 증감 연산 혹은 문자열로 표현 가능한 모든 자료 저장하는 목적으로 활용
        - 주요 명령어
            - 수정 : set, setnx,...
            - 조회 : get, mget
            - INCR 계열
    - Sets
        - 저장된 원소의 Unique 함을 보장하는 자료구조
        - Set 은 정렬을 보장하지 않음
        - Set 자료구조를 위한 연산 지원 ( 교집합 / 합집합 / 차집합 등 )
        - 주요 명령어
            - 수정 계열 : sadd, smove
            - 조회 계열 : smembers, scard, ..
            - 제거 계열 : spop, srem
            - 집합 계열 : sunion, sinter, sdif, ..
    - Sorted Sets
        - set + score 가중치 필드가 추가된 자료구조
        - 데이터 저장시, score 기반 오름차순 정렬로 저장됨
        - score 값이 같은 경우, 사전순으로 정렬
        - 주요 명령어
            - 수정 계열 : ZADD
            - 조회 계열 : ZRANGE, ZRANGEBYSCORE, ZRANK, ZSCORE, ..
            - 제거 계열 : ZPOPMIN, ZPOPMAX, ZREM, ..
            - 증감 계열 : ZINCRBY
            - 집합 계열 : ZUNIONSTORE, ZINTERSTORE
