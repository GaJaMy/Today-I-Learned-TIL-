# 클라우드 컴퓨팅

## 클라우드 컴퓨팅
```
서버, 스토리지, 데이터베이스, 네트워크 같은 IT 자원을 인터넷을 통해 필요한 만큼 빌려 사용하는 것

직접 서버 장비를 구입하지 않고, 필요할 때 생성해서 사용하고 사용량에 따라 비용을 냄
```

## On-Premiss VS 클라우드

### On-Premises
- 회사가 서버, 네트워크, 데이터베이스 등의 장비를 직접 구매하고 회사 내부 또는 자체 데이터센터에서 운영하는 방식이다.
- 장점
    - 서버와 데이터를 직접 관리할 수 있다.
    - 보안 정책이나 시스템 환경을 원하는 방식으로 세밀하게 구성할 수 있다.
    - 장기간 일정한 규모로 운영한다면 비용을 예측하기 쉽다.
    - 외부 클라우드 서비스 장애의 영향을 받지 않는다.
- 장점
    - 서버 장비 구매에 큰 초기 비용이 필요하다.
    - 서버 설치와 증설에 시간이 오래 걸린다.
    - 서버, 네트워크, 전력, 보안 등을 직접 관리해야 한다.
    - 사용량을 정확히 예측하지 못하면 장비가 부족하거나 낭비될 수 있다.
    - 장애에 대비한 별도의 데이터센터와 장비를 직접 준비해야 한다.

### 클라우드
- AWS와 같은 클라우드 사업자가 제공하는 서버, 스토리지, 데이터베이스 등의 IT 자원을 인터넷을 통해 빌려 사용하는 방식이다.
- 장점
    - 서버 장비를 직접 구매하지 않아도 된다.
    - 필요한 서버를 빠르게 생성할 수 있다.
    - 사용량에 따라 서버를 늘리거나 줄일 수 있다.
    - 사용한 만큼 비용을 지불할 수 있다.
    - 여러 Region과 Availability Zone을 이용해 고가용성을 구성하기 쉽다.
    - 데이터베이스, 모니터링, 메시지 큐 등 다양한 관리형 서비스를 사용할 수 있다.
- 단점
    - 사용량을 제대로 관리하지 않으면 비용이 크게 증가할 수 있다.
    - 클라우드 서비스와 네트워크 장애의 영향을 받을 수 있다.
    - AWS 서비스에 대한 별도의 학습이 필요하다.
    - 특정 클라우드 서비스를 많이 사용하면 다른 환경으로 이전하기 어려울 수 있다.
    - 서버와 데이터의 물리적인 장비를 직접 통제할 수 없다.

### 간단한 비교

| 구분    | 온프레미스         | 클라우드                |
| ----- | ------------- | ------------------- |
| 서버 소유 | 회사가 직접 소유     | 클라우드 사업자에게 빌려 사용    |
| 초기 비용 | 높음            | 상대적으로 낮음            |
| 서버 생성 | 장비 구매와 설치 필요  | 몇 분 내 생성 가능         |
| 확장성   | 장비를 추가 구매해야 함 | 필요에 따라 빠르게 확장 가능    |
| 운영 관리 | 회사가 직접 관리     | 일부 영역을 클라우드 사업자가 관리 |
| 비용 방식 | 장비 구매 중심      | 사용량 기반 비용           |
| 장애 대비 | 직접 이중화 환경 구성  | 여러 AZ와 Region 활용 가능 |
| 통제 범위 | 높음            | 상대적으로 제한적           |

온프레미스와 클라우드 중 항상 하나가 더 좋은 것은 아니다.

시스템의 보안 요구사항, 비용, 사용자 규모, 확장 가능성, 운영 인력 등을 고려하여 적절한 방식을 선택해야 한다.

일부 시스템은 온프레미스에서 운영하고 일부 시스템은 클라우드에서 운영하는 방식을 **하이브리드 클라우드(Hybrid Cloud)**라고 한다.
