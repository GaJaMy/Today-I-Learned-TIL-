# 웹 서비스
- 네트워크를 통해 서로 다른 기기 및 운영체제(OS) 간의 상호작용을 지원하는 소프트웨어 시스템
## 웹 서비스의 구조
- 다이어 그램
    ```plantuml
        @startuml
        left to right direction

        actor 사용자
        rectangle "프론트엔드" as Front
        rectangle "백엔드 서버" as Server
        database "DB" as DB

        사용자 --> Front : 화면 조작
        Front --> Server : request
        Server --> Front : response
        Server --> DB : query
        DB --> Server : query 결과
        @enduml
    ```

- 사용자에게 보여지는 영역 : 사용자 + 프론트 엔드
- 사용자에게 보여지지 않은 영역 :  백엔드 서버 + DB
- 사용자는 백엔드 서버, DB 서버에 직접 접근 불가능

## 역할
### 프론트 엔드
- 사용자에게 UI를 통해서 백엔드 서버에서 받은 결과를 시각화
- 백엔드 서버에 요청(request)를 통해 DB의 데이터를 요청
- 요청(request)에 대한 응답(response) 데이터를 시각화
### 백엔드 서버
- 프론트 엔드에서 들어온 요청을 바탕으로 알맞은 데이터를 DB에서 가져와서 프론트에 전달
- DB에 query를 통해서 데이터를 요창
- 비즈니스 로직에 따라 데이터를 정제 후, 프론트와 약속한 형태로 데이터 전달
### DB
- 웹 서비스에서 제공하는 데이터들이 저장되어있는 장소
- 쿼리를 통해 필요한 정보를 조회하거나, 저장할 수 있음

### API(Application Programing Interface)
- 프론트와 백엔드 사이의 데이터를 주고 받기 위한 규칙

## 웹 서비스의 흐름
- 흐름도
    ```plantuml
        @startuml
        left to right direction

        rectangle "사용자" as User
        rectangle "브라우저" as Browser
        rectangle "DNS 서버" as DNS
        rectangle "Web 서버" as WebServer
        rectangle "WAS 서버" as WebApplicationServer
        database "DB" as DB

        User --> Browser : URL 입력
        Browser --> User : Web 서버에서 받은 HTML 랜더링
        Browser --> DNS : URL 해당하는 IP 주소 요청
        DNS --> Browser : URL에 맞는 IP 주소 응답
        Browser --> WebServer : 정적 자원 요청
        WebServer --> Browser : 정적 자원 응답
        WebServer --> WebApplicationServer : 동적 자원 요청
        WebApplicationServer --> WebServer : 동적 자원 응답
        WebApplicationServer --> DB : 쿼리
        DB --> WebApplicationServer : 쿼리 결과
        @enduml
    ```

## 서버 처리 단계
- Web Server VS Web Application Server
    - 웹 서버 
        - 클라이언트의 HTTP 요청을 받아 HTML, CSS, JavaScript, 이미지와 같은 정적 리소스를 전달
    - Web Application Server
        - 클라이언트의 HTTP 요청을 받아 동적 리소스를 전달
    - WAS는 Web 서버를 포함 할수 도 있다.
- SSR(Server Side Rendering)
    - 서버가 데이터 조회와 HTML 생성을 수행한 뒤, 내용이 포함된 완성된 HTML을 브라우저에 전달한다. (전통적인 SSR 서버, Spring MVC + Thymeleaf)
    - 서버를 프론트 서버와, 백엔드 API 서버를 분리할 수도 있음, 대표적으로 Next.js 
- CSR(Clinet Side Rendering)
    - 브라우저가 먼저 HTML, CSS, JavaScript와 같은 정적 파일을 전달받는다.
    - 이 정적 파일은 반드시 백엔드 API 서버에서 받을 필요는 없으며, 다음과 같은 서버에서 제공될 수 있다.
    - Nginx, CDN, AWS S3, Vercel, Netlify, 별도의 프론트엔드 서버

## ORM
- 객체의 관계형 데이터 베이스를 매핑 하는 기술
    - JPA, MyBatice 등등
- DB와 WAS 사이에서 데이터를 주고 받기 위함

## DB
- 데이터를 CRUD하는 단계
- ORM이 실행한 SQL 쿼리를 처리 후 결과 반환
